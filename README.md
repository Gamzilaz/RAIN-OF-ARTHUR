# 🌧️ Документация эффекта "Rain on Glass WebGL" (Физическое преломление дождя на стекле)

Этот документ содержит исчерпывающее руководство по архитектуре, математике, шейдерам, звуковому синтезу и логике грозы/циклов погоды для проекта **Rain on Glass WebGL**. 

Файл разработан как **"Максимальный Промт-Протокол"**, который можно передать любому другому AI-агенту или разработчику для воссоздания этого потрясающего эффекта один в один на фоне любого другого веб-приложения.

---

## 🗺️ 1. Обзор архитектуры системы (Cascade Pipeline)

Эффект построен по принципу многоуровневого каскадного конвейера:
1. **React UI (Слой управления):** Рендерит интерфейс панели настроек, считывает ползунки, управляет состояниями файлов, переключает пресеты и координирует фазы погодных циклов.
2. **Simulation Engine (Движок симуляции - HTML5 Canvas 2D):** Рассчитывает физику капель на невидимом холсте в реальном времени. Генерирует карту нормалей (нормали водной поверхности) и карту блеска/бликов.
3. **Rendering Pipeline (Конвейер рендеринга - WebGL):** Считывает текстуры из Canvas 2D, накладывает фоновое изображение, рассчитывает физическое преломление света (рефракцию) во фрагментном шейдере (`Water.frag`) и отрисовывает финальный кадр с учетом параллакса и вспышек молний.
4. **Procedural Audio Synthesizer (Аудио-синтезатор):** Процедурно генерирует шум ветра, звуки падающих капель и раскаты грома через Web Audio API совместно с воспроизведением низкочастотных звуковых сэмплов грозы.

```
+------------------------------------------------------------+
|                        React App UI                        |
|   (Пресеты, циклы погоды, таймеры, загрузка своего фона)   |
+-----------------------------+------------------------------+
                              | Координация и параметры
                              v
+------------------------------------------------------------+
|             Simulation Engine (Canvas 2D Buffer)           |
|  - raindrops.ts: физика капель, коллизии, ветер, сползание |
|  - texture-generator.ts: генерация карт нормалей и блеска  |
+-----------------------------+------------------------------+
                              | Текстуры u_waterMap & u_textureShine
                              v
+------------------------------------------------------------+
|                 WebGL Rendering Pipeline                   |
|  - Simple.vert: вершинный шейдер (простая сетка)           |
|  - Water.frag: фрагментный шейдер преломления (рефракции)   |
|  - Отрисовка фонового изображения с параллаксом и молнией  |
+------------------------------------------------------------+
```

## 📂 2. Картография файлов проекта

При переносе эффекта в другой проект вам потребуются следующие файлы из репозитория:

## 🌧️ 3. Физика симуляции капель (2D Canvas)

Физическое поведение капель симулируется в реальном времени в файле `raindrops.ts`. Капли делятся на два типа:
- **Статические микрокапли (Droplets):** Мелкий конденсат на стекле. Они не двигаются, но постепенно испаряются или поглощаются крупными каплями при пересечении.
- **Динамические капли (Raindrops):** Крупные капли, на которые действует сила гравитации, заставляя их скользить вниз, оставляя за собой след (trail) из мелких капелек.

### Основные физические законы симулятора:
1. **Гравитационное сползание (Glide):** Крупная капля скользит вниз, если ее масса (радиус) превышает определенный порог. Скорость сползания пропорциональна весу капли и умножается на `dropFallMultiplier`.
2. **Влияние ветра (`windVelocity`):** Ветер смещает вектор падения капель по горизонтальной оси `X`. Капли сползают под углом, зависящим от силы и направления ветра.
3. **Механика следа (Trail):** При движении динамическая капля теряет массу (радиус уменьшается на `trailRate`), оставляя позади себя цепочку мелких статических капель. Если радиус динамической капли падает ниже минимального предела (`minR`), она превращается в статическую и останавливается.
4. **Слияние при коллизиях (Collision & Coalescence):** Если скользящая капля пересекается с другой каплей на расстоянии ближе, чем `collisionRadius`, они сливаются. Масса поглощенной капли переходит к движущейся, увеличивая ее радиус и скорость, а поглощенная капля удаляется из симуляции.
5. **Испарение (`evaporationRate`):** Со временем капли медленно уменьшаются в размере под действием испарения, предотвращая переполнение экрана водой.

## 🎨 4. WebGL рендеринг и шейдерное преломление

Секрет фотореализма кроется во фрагментном шейдере `Water.frag`. Вместо отрисовки плоских капель, WebGL преломляет фоновое изображение на основе геометрии капли.

### Как работает рефракция (Water.frag):
1. **Карта воды (`u_waterMap`):** Текстура, сгенерированная из Canvas симуляции. В ней закодированы нормали поверхности. Красный канал `R` содержит смещение по оси `X`, зеленый канал `G` — смещение по оси `Y`, а синий канал `B` — маску блеска/бликов.
2. **Расчет вектора смещения:** Шейдер считывает пиксель из `u_waterMap`. Если значение цвета отличается от нейтрального (воды нет), шейдер рассчитывает вектор отклонения светового луча:
   $$\text{offset} = (\text{normal.rg} - vec2(0.5)) \times \text{u\_refractionStrength}$$
3. **Преломление фона:** Шейдер сдвигает текстурные координаты фонового изображения (`u_textureBg`) на полученный вектор смещения. В результате фон внутри капли отображается перевернутым и искаженным, в точности как сквозь сферическую линзу настоящей капли воды!
4. **Тени от капель (`u_renderShadow`):** Для придания объема шейдер рассчитывает искусственную тень на границах капель, сдвигая маску капли по вектору освещения и вычитая ее из яркости фона.
5. **Динамический параллакс:** На основе смещения мыши или гироскопа (`u_parallax`), шейдер независимо сдвигает координаты фонового изображения (`u_parallaxBg`) и карты капель (`u_parallaxFg`), создавая иллюзию глубины: как будто стекло с каплями находится на расстоянии от размытого заднего плана.

## ⛈️ 5. Детальная логика грозы, молний и грома (Deep Dive)

Система грозы полностью синхронизирована по визуальным и аудио-каналам. Она работает в двух режимах: **автоматическом** (по вероятностям) и **синематическом** (регламентированном фазами цикла).

### 5.1. Разделение типов грозы:
- **Далекий рокот (Distant Rumble - `groza1`):** Воспроизводит только низкочастотный глухой звук раската грома без визуальной вспышки. Показывает, что шторм бушует где-то вдали.
- **Прямой удар молнии (Direct Strike - `groza2`):** Сопровождается мгновенной ослепительной вспышкой экрана, прорисовкой электрического разряда на стекле и мощным, раскатистым звуком близкого удара.

---

### 5.2. Визуальное построение молнии (Фрактальный SVG-путь):
Когда срабатывает триггер вспышки, функция `generateLightning` генерирует реалистичный ветвящийся путь молнии.
1. **Основной ствол (Main Path):** Начинается на случайной координате `X` вверху экрана ($Y \approx 0$) и движется к нижней границе зигзагообразными сегментами с фрактальным смещением (джиттером).
2. **Боковые ветви (Branches):** С вероятностью 30% от каждого сустава основного ствола может отпочковаться боковая ветвь, которая тоньше основного ствола и затухает по мере продвижения в сторону.
3. **Отрисовка:** Молния рендерится как динамический SVG-элемент поверх WebGL холста с использованием неонового свечения:
   ```css
   filter: drop-shadow(0px 0px 8px rgba(165, 180, 252, 0.9))
   ```

---

### 5.3. Физика затухания вспышки в WebGL:
Молния не гаснет мгновенно. Ее свет ионизирует воздух, оставляя послесвечение.
- Для имитации этого процесса используется **массив кадров затухания (Decay Frames)**.
- После окончания фазы активных ударов создается от 30 до 50 дополнительных микро-шагов с интервалом в 15 мс.
- На каждом шаге яркость WebGL-фона (`u_brightness`) и прозрачность SVG-канала (`opacity`) плавно умножаются на коэффициент затухания:
  $$\text{brightnessShift} = \text{brightnessShift} \times 0.92$$
  $$\text{opacity} = \text{opacity} \times 0.90$$
- Это создает мягкий, обволакивающий, фотореалистичный спад свечения (decline tail), длящийся около 500-800 мс.

---

### 5.4. Типы разрядов (Strike Patterns):
При активации прямого удара случайным образом выбирается один из трех паттернов:

#### 1. Одиночный удар (Single Strike - 55% шанс):
- Самый простой разряд.
- 1 резкий пик яркости ($2.85 \times \text{интенсивность}$), за которым следует мгновенное плавное затухание.
- Используется один сгенерированный SVG-путь.

#### 2. Двойной удар (Double Strike - 35% шанс):
- Имитирует классическую молнию с повторным разрядом по тому же ионизированному каналу.
- **Фаза 1:** Первичный мощный импульс (яркость $2.85 \times \text{интенсивность}$). Плавно тухнет в течение 3 кадров до 15%.
- **Фаза 2 (Микро-пауза):** Полная темнота на 2 кадра (30 мс), SVG-путь скрывается.
- **Фаза 3:** Вторичный удар (яркость $2.35 \times \text{интенсивность}$) по новому, слегка смещенному SVG-каналу в той же области экрана.
- **Фаза 4:** Финальный спад свечения до нуля.

#### 3. Тройной удар (Triple Strike - 10% шанс):
- Редкое, крайне эффектное зрелище из трех каскадных разрядов подряд.
- **Импульс 1:** Первый удар (яркость $2.85$), спад до 15% за 3 кадра.
- **Пауза 1:** Темнота на 2 кадра (30 мс).
- **Импульс 2:** Второй удар средней мощности (яркость $2.15$), спад до 12% за 3 кадра.
- **Пауза 2:** Темнота на 2 кадра (30 мс).
- **Импульс 3:** Третий, завершающий мощный разряд (яркость $2.65$) с последующим глубоким затуханием (30-50 кадров).

---

### 5.5. Математическая модель вспышки в шейдере (`Water.frag`):
При вспышке WebGL окрашивает бэкграунд не просто в белый цвет, а производит аддитивное смешивание с глубоким электрическим фиолетовым спектром:
```glsl
// Электрический фиолетово-синий оттенок вспышки
vec3 electricViolet = vec3(0.55, 0.60, 1.0); 
vec3 finalFlash = u_flashColor * electricViolet * delta;
gl_FragColor.rgb += finalFlash;
```
Это придает молнии "магический", неоновый характер свечения, заставляя мокрые ветви леса на заднем плане озаряться реалистичным грозовым светом.

## ⏳ 6. Движок погодных циклов (Weather Cinematic Cycles)

Когда включена опция **"Запустить цикл погоды" (Weather Cycle)**, приложение переходит в режим бесконечного кинематографического изменения погоды. Цикл состоит из 5 последовательных этапов:

```
[Phase 1: Clear Mist] ---> [Phase 2: Drizzle] ---> [Phase 3: Active Rain]
        ^                                                   |
        |                                                   v
[Phase 5: Clear Mist] <--- [Phase 4: After-Storm] <--- [Phase 4: Thunderstorm]
```

### Фазы цикла и их характеристики:
1. **Туманная просека (Clear Mist - 15 сек):** Дождя почти нет (`rainChance` = 0.02), ветер слабый. Нежный туман.
2. **Легкая морось (Drizzle - 18 сек):** Появление первых мелких капель (`rainChance` = 0.15). Звук дождя начинает плавно нарастать.
3. **Ливень (Active Rain - 20 сек):** Полноценный затяжной дождь (`rainChance` = 0.40). Интенсивный шум воды. **Срабатывает ровно 1 кинематографическая вспышка молнии** ближе к концу фазы.
4. **Эпицентр бури (Thunderstorm - 25 сек):** Максимальный шторм. Ливень стеной (`rainChance` = 0.55), ураганный ветер. **Срабатывает ровно 2 крупных удара молнии**, случайно распределенных по времени.
5. **Затухание бури (After-Storm - 15 сек):** Дождь стихает до мороси, ветер утихает, гроза уходит.

### Плавный морфинг параметров (Параметрический Lerp):
Переход между фазами происходит абсолютно бесшовно. Все физические параметры капель (шансы, размеры, скорость сползания, скорость ветра) и громкость звуков интерполируются каждую секунду с использованием функции линейной интерполяции (`lerp`) и функции сглаживания `ease-in-out`:
$$t_{\text{eased}} = t^2 \times (3 - 2t)$$
$$\text{Param}_{\text{current}} = \text{lerp}(\text{Param}_{\text{start}}, \text{Param}_{\text{target}}, t_{\text{eased}})$$
